**Pick A Flick V1.3**

Repo cleanup plus major spin-wheel UX upgrades since v1.2.

**Highlights**

- **Project layout** — App moved to `app/` (`index.html`, `main.js`, `style.css`); generated assets in `app/generated/`; live UI art in `graphics/ui/`; Tix Mix title cards in `graphics/tix-mix/`; docs in `docs/`; audio batch shortcuts in `scripts/shortcuts/`.
- **Two-stage spin button** — Press-and-release interaction (not single-click): separate button-down / release audio clips, visual depressed state, spin starts on release.
- **Reveal modes** — New **Reveal Mode** toggle cycles **Classic** (pause after landing), **Snappy** (instant reveal; quick tap = short spin, hold longer = longer spin with ×1.3–×3.6 exponential ramp up to 10s), and **Speed Run** (1s spin, skips intro, win clip on button release).
- **Snappy power gauge** — Animated `#spin-power-gauge` fills while the spin button is held in Snappy mode.
- **Undead wheel toggle** — Switch between default and alternate wheel disc artwork.
- **Watched list file linking** — File System Access API support to link and save directly to `app/watched-movies.js`.
- **Docs and agent workflow** — `docs/AGENTS.md`, `docs/USER-GUIDE.md`, category/focus docs, 8 Cursor rules, section banners in `main.js`, `.cursorignore` for staging/archive paths.

**After upgrading**

- Dev server entry point is now `http://127.0.0.1:8765/app/index.html` (root `/` redirects).
- Run `npm run refresh-audio` if spin-button clips do not play after pull.
